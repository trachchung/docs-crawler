## Manage Payments
Let's look at some of the features available for managing a payment object.
#### [Creating a payment objectCopied!](https://docs.tryspeed.com/docs/receive-payments/payment/payments-manage#creating-a-payment-object)
You can directly create a payment object using the payment APIs. Also, a payment object is automatically generated for each customer's checkout session, payment link, or checkout link.
#### [Retrieve a PaymentCopied!](https://docs.tryspeed.com/docs/receive-payments/payment/payments-manage#retrieve-a-payment)
By utilizing the retrieve payment API, you can access comprehensive details of a specific payment object. The payment payload displayed corresponds to the payment ID you provide. Employ the `GET` [/payments/{id}](https://apidocs.tryspeed.com/reference/payment-retrieve) `API` to retrieve the payload of the specified payment.
#### [List all PaymentsCopied!](https://docs.tryspeed.com/docs/receive-payments/payment/payments-manage#list-all-payments)
Through the list all payment APIs, you can view a list of all payment objects associated with checkouts, payment links, and checkout sessions. To obtain all the payment IDs and the objects associated with them, you must use the `GET` [/payments/](https://apidocs.tryspeed.com/reference/payment-list)`API`.
#### [Filter all PaymentsCopied!](https://docs.tryspeed.com/docs/receive-payments/payment/payments-manage#filter-all-payments)
The Filter All Payments API enables you to refine payments using parameters like payment ID, status, currency, target currency, amount, target amount, target amount paid, and other relevant criteria. This functionality assists in pinpointing the specific payment you seek.
Please refer `POST` [/payments/filter](https://apidocs.tryspeed.com/reference/payment-filter) `API` for more details.
#### [Search PaymentsCopied!](https://docs.tryspeed.com/docs/receive-payments/payment/payments-manage#search-payments)
Utilize this API to filter payments using query, page, and limit parameters. The Search Payments API intends to fetch payments based on Speed's query language. Employ the **`POST`**[/search/payments](https://apidocs.tryspeed.com/reference/search-payments) `API` for conducting payment searches.
#### [Cancelling a payment objectCopied!](https://docs.tryspeed.com/docs/receive-payments/payment/payments-manage#cancelling-a-payment-object)
You can cancel a payment if you no longer wish to accept any payments via it. If you want to cancel a payment, you must do so before it is paid or expires.
As of now, you can cancel a payment object associated with the checkout link or payment link by deactivating the link through APIs or the Speed web application. You can also cancel a payment associated with a checkout session through the checkout session APIs.
Below are the payment API endpoints used for the creation of payments, retrieving payments by ID, fetching all payments collectively, applying filters, and conducting payment searches.
`POST` [Create a Payment](https://apidocs.tryspeed.com/reference/payment-create) `API`
`GET` [Retrieve a Payment](https://apidocs.tryspeed.com/reference/payment-retrieve) `API`
`GET` [List all payments](https://apidocs.tryspeed.com/reference/payment-list) `API`
`POST` [Filter all payments](https://apidocs.tryspeed.com/reference/payment-filter) `API`
`POST` [Search payments](https://apidocs.tryspeed.com/reference/search-payments) `API`
#### [Track paymentsCopied!](https://docs.tryspeed.com/docs/receive-payments/payment/payments-manage#track-payments)
You can check the payment outcome on your web application, which provides detailed insights. Alternatively, you can configure notification webhooks to receive payment updates. Learn more about [Webhook subscriptions.](https://apidocs.tryspeed.com/reference/webhooks-and-events)
